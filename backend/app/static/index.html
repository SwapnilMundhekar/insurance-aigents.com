Agentic RAG platform for insurance operations.

InsuranceAIGents.com is a production-style AI engineering project that demonstrates governed multi-agent orchestration, policy-grounded retrieval, tool calling, memory, evaluation, audit trails, and observability for insurance workflows.

GitHub repository

What this project proves.

This is not a basic chatbot. It is an end-to-end Agentic RAG system designed to show the skills expected from an AI engineering role in insurance.

Problem

Insurance teams need AI systems that can answer policy and claims questions without guessing, leaking sensitive data, ignoring guardrails, or making unsupported claim decisions.

Solution

A governed Agentic RAG workflow that checks the prompt, retrieves policy evidence, calls approved tools, reflects on the answer, validates output safety, and logs the full trace.

Local-first build

The system runs on a local PC using FastAPI, Ollama, document ingestion, chunking, embeddings, memory fallback, audit logs, and observability endpoints before cloud deployment.

End-to-end flow

1. User asks an insurance question through API or chatbot
2. InputGuardrailAgent checks domain, intent, prompt injection, and personal data
3. QueryRewriteAgent rewrites the question for retrieval
4. RetrievalAgent searches indexed insurance document chunks
5. ToolCallingAgent calls coverage, evidence, fraud, and human review tools through MCP bridge
6. AnswerAgent generates a grounded decision-support response
7. ReflectionAgent critiques reliability and risk
8. OutputGuardrailAgent validates the final answer before returning it

Capabilities.

These are the same capabilities that should be highlighted in the GitHub README and interview explanation.

Governance

Business domain routing, intent classification, prompt injection blocking, personal data redaction, and allowed tool mapping.

Document intelligence

PDF readability checks, OCR decision layer, semantic chunking, page metadata, section detection, and source traceability.

Agentic RAG

Supervisor workflow with query rewriting, retrieval, tool calling, answer generation, reflection, and output validation.

MCP tool bridge

Approved tools are exposed through an MCP-style bridge for coverage checks, evidence requirements, fraud signals, and human review decisions.

Memory and audit

Session working memory, SQLite audit persistence, workflow traces, tool call history, retrieval sources, and review status.

Evaluation and observability

Evaluation cases for guardrails and output safety, observability summaries, trace analytics, risk counts, and tool usage metrics.
